Yavela | Documentation

Web API Base URL https://api.yavela.xyz

Errors

All error responses include success: false and a message field describing the issue.

200OK — Request succeeded.
400Bad Request — Missing or invalid parameters.
401Unauthorized — Invalid or missing API key.
404Not Found — Version or resource not found on CDN.
500Internal Server Error — Something went wrong on our end.

Roblox Version

Returns the latest deployed Roblox client versions for Windows and macOS, along with deployment timestamps.

GET /roblox/version Get latest Roblox versions

No parameters required.

Response
{ "success": true, "windows_version": "version-ae421f0582e54718", "windows_date": "3/23/2026, 6:51:30 PM UTC", "mac_version": "version-0b2afd6436e74a1f", "mac_date": "3/23/2026, 6:51:30 PM UTC" }
cURL
curl https://api.yavela.xyz/roblox/version

FFlags

Returns the full Roblox Fast Flags dataset as key-value pairs. The response payload is large — cache it and refresh periodically.

GET /roblox/fflags Get all Roblox FFlags
Response is very large. Do not call on every request — cache the result on your end.

No parameters required. Returns all flags as string key-value pairs.

Partial Response
{ "FFlagDebugGraphicsPreferVulkan": "False", "DFFlagTaskSchedulerLimitTargetFpsTo2402": "True", "FIntRenderShadowIntensity": "75" }
cURL
curl https://api.yavela.xyz/roblox/fflags

Obfuscate

Obfuscates a Lua script. Send the script source as a JSON body and receive an obfuscated version in return.

POST /api/obfuscate Obfuscate a Lua script
Body Parameters
NameTypeRequiredDescription
script string required Lua script source to obfuscate. Send as application/json.
cURL
curl -X POST https://api.yavela.xyz/api/obfuscate \ -H "Content-Type: application/json" \ -d '{ "script": "print(\"hello world\")" }'

Deploy Check

Verifies whether a specific Roblox version exists on the CDN and returns its package manifest.

GET /api/roblox/deploy-check Check if a version exists on CDN
Query Parameters
NameTypeRequiredDescription
version string required Version string to check. The version- prefix is optional.
platform string optional WindowsPlayer (default) or MacPlayer.
Found Response
{ "success": true, "version": "version-ae421f0582e54718", "platform": "WindowsPlayer", "exists": true, "manifest_url": "https://setup-aws.rbxcdn.com/version-ae421f0582e54718-rbxPkgManifest.txt", "package_count": 12, "packages": ["RobloxApp.zip", "Libraries.zip", ] }
Not Found Response
{ "success": false, "message": "Version not found on CDN", "version": "version-ae421f0582e54718", "platform": "WindowsPlayer" }
cURL
curl "https://api.yavela.xyz/api/roblox/deploy-check?version=ae421f0582e54718&platform=WindowsPlayer"

App API Overview

The Yavela App API is a .NET assembly (yavAPI.dll) with the namespace yavAPI and a single class FUNC that exposes functions for controlling the executor from your own WinForms or WPF application.

Requirements: .NET Framework 4.8 only. Supports WinForms and WPF. The DLL assembly and all dependencies are available in the Yavela Discord server.
Setup (C#)
// 1. Add yavAPI.dll as a reference in your project // 2. Import the namespace using yavAPI;
All Functions
yavAPI.FUNC.Execute(string script); yavAPI.FUNC.Attach(); yavAPI.FUNC.SetAutoAttach(bool enabled); yavAPI.FUNC.KillRoblox();

Execute

Executes a Lua script string in the attached Roblox process. Call Attach() first unless auto-attach is on.

Signature
yavAPI.FUNC.Execute(string script);
ParameterTypeRequiredDescription
script string required Lua script to execute. Typically passed from a textbox in your UI.
WinForms / WPF Example
private void btnExecute_Click(object sender, EventArgs e) { yavAPI.FUNC.Execute(scriptTextBox.Text); } private void BtnExecute_Click(object sender, RoutedEventArgs e) { yavAPI.FUNC.Execute(ScriptBox.Text); }

Attach

Attaches the executor to the running Roblox process. Must be called before Execute() unless SetAutoAttach(true) is active.

Signature
yavAPI.FUNC.Attach();
WinForms / WPF Example
private void btnAttach_Click(object sender, EventArgs ) { yavAPI.FUNC.Attach(); } private void BtnAttach_Click(object sender, RoutedEventArgs e) { yavAPI.FUNC.Attach(); }

SetAutoAttach

Enables or disables automatic attachment when Roblox launches. When enabled, you do not need to call Attach() manually.

Signature
yavAPI.FUNC.SetAutoAttach(bool enabled);
ParameterTypeRequiredDescription
enabled bool required true to enable auto-attach, false to disable.
WinForms / WPF Example
private void chkAutoAttach_CheckedChanged(object sender, EventArgs e) { yavAPI.FUNC.SetAutoAttach(chkAutoAttach.Checked); } private void ChkAutoAttach_Checked(object sender, RoutedEventArgs e) { yavAPI.FUNC.SetAutoAttach(true); } private void ChkAutoAttach_Unchecked(object sender, RoutedEventArgs e) { yavAPI.FUNC.SetAutoAttach(false); }

KillRoblox

Terminates the running Roblox process immediately.

Signature
yavAPI.FUNC.KillRoblox();
WinForms / WPF Example
private void btnKill_Click(object sender, EventArgs e) { yavAPI.FUNC.KillRoblox(); } private void BtnKill_Click(object sender, RoutedEventArgs e) { yavAPI.FUNC.KillRoblox(); }

Velocity API Overview

The Velocity API is a modern .NET 10.0 assembly exposing the yavAPI.Velocity class. It provides a streamlined interface for injection, script execution, and Roblox process management.

Requirements: .NET 10.0 only. The DLL assembly and all dependencies are available in the Yavela Discord server.
Setup (C#)
// 1. Add yavAPI.dll as a reference in your project // 2. Import the namespace using yavAPI;
All Functions
yavAPI.Velocity.InjectAPI(); yavAPI.Velocity.Execute(string scriptSource); bool injected = yavAPI.Velocity.IsInjected(); bool open = yavAPI.Velocity.IsRobloxOpen(); yavAPI.Velocity.KillRoblox();

InjectAPI

Injects the Yavela API into the running Roblox process. Must be called before Execute(). Use IsRobloxOpen() to confirm Roblox is running before calling this.

Signature
yavAPI.Velocity.InjectAPI();
Example
private void btnInject_Click(object sender, EventArgs e) { if (yavAPI.Velocity.IsRobloxOpen()) yavAPI.Velocity.InjectAPI(); }

Execute

Executes a Lua script string in the injected Roblox process. Ensure InjectAPI() has been called and IsInjected() returns true first.

Signature
yavAPI.Velocity.Execute(string scriptSource);
ParameterTypeRequiredDescription
scriptSource string required Lua script source to execute in the Roblox process.
Example
private void btnExecute_Click(object sender, EventArgs e) { if (yavAPI.Velocity.IsInjected()) yavAPI.Velocity.Execute(scriptTextBox.Text); }

IsInjected

Returns whether the API is currently injected into the Roblox process.

Signature
bool result = yavAPI.Velocity.IsInjected();
returns bool true if injected, false otherwise.
Example
private void UpdateStatus() { bool injected = yavAPI.Velocity.IsInjected(); statusLabel.Text = injected ? "Injected" : "Not Injected"; btnExecute.Enabled = injected; }

IsRobloxOpen

Returns whether a Roblox process is currently running.

Signature
bool result = yavAPI.Velocity.IsRobloxOpen();
returns bool true if Roblox is running, false otherwise.
Example
private void timer_Tick(object sender, EventArgs e) { btnInject.Enabled = yavAPI.Velocity.IsRobloxOpen(); }

KillRoblox

Terminates the running Roblox process immediately.

Signature
yavAPI.Velocity.KillRoblox();
Example
private void btnKill_Click(object sender, EventArgs e) { yavAPI.Velocity.KillRoblox(); }
